## Overview

Request Network contracts are grouped into three categories:

- **Storage contracts**: anchor IPFS CIDs and storage-related state
- **Payment contracts**: process payment flows across supported chains
- **REQ token and burn mechanism contracts**: support protocol token economics

## Storage contracts

Core storage contracts include:

- `RequestOpenHashSubmitter`
- `StorageFeeCollector`
- `RequestHashStorage`

These contracts handle CID declarations and storage-fee-related flows.

## Payment contracts

Payment contracts include proxies and advanced payment primitives such as:

- `ERC20FeeProxy`
- `EthereumFeeProxy`
- `ERC20EscrowToPay`
- `BatchConversionPayments`
- `BatchNoConversionPayments`
- `ERC20SwapToPay`
- `ERC20ConversionProxy`
- `ETHConversionProxy`
- `ERC20SwapToConversion`
- `ERC20TransferableReceivable`

## Single request proxy contracts

For simplified payment flows:

- `ERC20SingleRequestProxy`
- `EthereumSingleRequestProxy`
- `SingleRequestProxyFactory`
